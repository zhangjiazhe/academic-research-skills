# Engineering Paper Literature Review Rules

1. Organize the review as a time-ordered or theory-progress-ordered research trajectory, not as a list of isolated cases.
2. Start with technical evolution: old methods became insufficient, new methods emerged, and the new methods still have limitations that this study aims to fill.
3. Clarify inheritance, critique, and innovation relationships among prior studies so the development path of the field is visible.
4. Extract the turning points, disputes, unresolved issues, and emerging trends in each category.
5. Explicitly connect the unresolved issues to the research questions and urgency of this study.
6. Distinguish the present work from prior work, especially in what is done differently or more effectively.
7. Do not present this paper's method before explaining why the problem requires that method.
8. When multiple solution types address the same problem, a comparison table may be used.
9. When writing the review, coordinate with the reference rules and citation-placement rules.
10. If the current source base is insufficient, outdated, or unverifiable, trigger an upstream `deep-research` literature search before continuing the review.
11. Do not fabricate or guess references in order to complete the review structure.
