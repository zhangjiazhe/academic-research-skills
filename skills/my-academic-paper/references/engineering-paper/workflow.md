# Engineering Paper Workflow

## Core Workflow

1. Apply these rule sets throughout the whole paper:
   - Title rules
   - References rules
   - Formatting rules
   - Figure, table, and formula rules
   - Citation placement rules
   - Wording rules
2. Follow this fixed section order:
   - Method
   - Results
   - Discussion
   - Conclusion
   - Future Work
   - Introduction
3. After the user explicitly says `开始写作`, draft continuously according to the approved overall framework.
4. Each major part must be prepared as a separate file, not merged into a single source file during drafting:
   - `method.*`
   - `results.*`
   - `discussion.*`
   - `conclusion.*`
   - `future_work.*`
   - `introduction.*`
5. The actual writing flow is:
   - First write Method
   - Then write Results
   - Then write Discussion based on the Results
   - Then write Conclusion based on all prior sections
   - Then write Future Work
   - Finally write Introduction
6. Distinguish internal drafting order from final paper presentation order:
   - Internal drafting order must remain: Method -> Results -> Discussion -> Conclusion -> Future Work -> Introduction
   - Final assembled paper must follow standard paper presentation order, with `Introduction` placed before `Method`
7. When assembling the final deliverable, present the major sections in this order:
   - Introduction
   - Method
   - Results
   - Discussion
   - Conclusion
   - Future Work
